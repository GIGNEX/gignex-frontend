# Contributing to Gignex Frontend

First off, thanks for taking the time to contribute! 🎉

## Development Workflow

1. **Fork & Clone**: Fork the repository and clone it locally.
2. **Install Dependencies**: Run `npm install` to get the necessary Node modules.
3. **Branching**: Create a new branch for your feature or bug fix: `git checkout -b feature/your-feature-name`.
4. **Development**:
   - Use `npm run dev` to start the local Vite server.
   - Adhere to the existing design system found in `src/index.css` and `src/App.css`.
   - Maintain the "holographic/deep-space" aesthetic for all new components.
5. **Testing**: Test your changes against both the mockup backend and the local Stellar testnet.
6. **Commit**: Write clear, concise commit messages.
7. **Pull Request**: Open a PR against the `main` branch, describing your changes in detail.

## Style Guide
- Use functional React components and hooks.
- Favor CSS variables (e.g., `var(--neon-teal)`) over hardcoded colors.
- Ensure all icons are pulled from `lucide-react`.

We look forward to your cosmic contributions! 🚀
